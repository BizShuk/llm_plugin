# `代理向量協定 (Agent Vector Protocol, AVP)`

這張內容提到的 `AVP` 全稱是 `代理向量協定 (Agent Vector Protocol)`，這是一個最近在 `人工智慧 (Artificial Intelligence, AI)` 開發者社群（如 `Threads`, `Reddit`）引起討論的新技術。

簡單來說，它改變了「多個 `人工智慧代理 (AI Agents)` 互相溝通」的方式。

### 什麼是 `鍵值快取 (Key-Value Cache, KV-cache)`？

在理解 `AVP` 之前，要先知道 `鍵值快取 (Key-Value Cache, KV-cache)` 是什麼：

1. `記憶片段`： 當 `AI` 讀取一段文字（例如 1000 字的背景資料）時，它會把這段文字轉換成複雜的數學矩陣（`鍵 (Key)` 和 `值 (Value)`），用來代表它對這些文字的「理解」或「注意力狀態」。
2. `重複利用`： 如果沒有快取，`AI` 每次回答新問題都要重新讀取那 1000 字，非常浪費效能。有了 `KV-cache`，`AI` 就直接翻閱剛才存好的「筆記」，跳過重新閱讀的過程，直接生成新字。

---

### `AVP` 做了什麼？(關鍵在於「傳遞記憶」)

傳統的 `AI` 代理溝通（如 `AutoGPT`, `LangChain`）是用 `文字 (Text)` 傳輸：

- `代理 A`： 寫下一段話（文字）。
- `代理 B`： 讀取這段話 $\rightarrow$ 重新計算 `權標 (Token)` $\rightarrow$ 產生自己的 `KV-cache` $\rightarrow$ 才開始思考。
- `缺點`： 每次交棒都要重新算一遍（稱為 `預填充 (Prefill)` 階段），文字越長，算力成本越高。

`代理向量協定 (Agent Vector Protocol, AVP)` 則是直接傳遞 `KV-cache`：

- `代理 A`： 把自己大腦裡的「記憶狀態（`KV-cache`）」直接打包發給 `代理 B`。
- `代理 B`： 把這包記憶「注入」到自己的腦中，完全不需要讀文字。
- `優點`： 跳過了最耗時、最貴的「重新閱讀」過程。

---

### 為什麼這能省錢又變快？

這就是內容中提到「`Token` 成本暴跌 78%」和「速度翻倍」的原因：

1. `省去重複算力`：
   在多代理流程中，通常有 70%-80% 的 `Token` 是重複的背景資訊。`AVP` 讓 `代理 B` 不需要為了這些重複資訊付錢給推理引擎（因為它根本沒「讀」那些 `Token`，是直接注入狀態）。
2. `跳過推理瓶頸`：
   `AI` 最慢的地方在於「理解長文本（`Prefill`）」。`AVP` 讓 `代理 B` 直接進入「寫答案（`解碼, Decode`）」階段，所以運行速度會變快 2 到 4 倍。

---

### 分析與拆解：`AVP` 的運作流程

若要實作這種機制，步驟通常如下：

- `步驟 1. 狀態序列化 (Serialization)`：
  `代理 A` 完成工作後，將其 `圖形處理器 (Graphics Processing Unit, GPU)` 記憶體中的 `KV-cache` 轉換成 `二進制 (Binary)` 格式。
- `步驟 2. 協定握手 (Handshake)`：
  確認 `代理 B` 是否使用相同的模型（例如都是 `Llama-3`）。因為不同模型對記憶的「格式」理解不同，目前主要支援同系列模型。
- `步驟 3. 記憶注入 (Memory Injection)`：
  `代理 B` 接收數據，直接載入快取，跳過對原始 `提示詞 (Prompt)` 的計算。
- `步驟 4. 直接推理 (Inference)`：
  `代理 B` 直接從最後一個 `Token` 開始產出內容。

### 生活化比喻

這就像是兩個人在傳球：

- `傳統方式`： 我寫了一張長長的清單（文字），你拿去讀完、記住，再把球傳給下一個人。
- `AVP 方式`： 我直接把我腦袋裡關於這張清單的「記憶記憶體」複製一份給你，你睜開眼就已經全記住了，不需要讀紙上的字。

`注意`： 目前這項技術主要應用在 `私有化部署 (Self-hosted)` 的模型（如 `vLLM` 框架），因為 `OpenAI` 或 `Claude` 的 `應用程式介面 (Application Programming Interface, API)` 通常不會開放底層的 `KV-cache` 讓使用者直接存取。
